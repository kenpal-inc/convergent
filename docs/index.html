convergent applies this to code: multiple AI personas with different priorities analyze the same problem, then a synthesizer merges what they agree on.

How It Works

Give it a codebase and a goal. It handles the rest.

Phase 0
🗒

Task Generation

Analyzes your codebase and goal, decomposes into a structured task queue with dependency graph. Assigns types: code, explore, or command.

Phase A
🧸

Convergence

3–7 personas plan in parallel with different priorities. Early termination on high consensus. Synthesizer merges the optimal plan.

Phase B
⌨

Execution

Executes the converged plan with cross-task learnings as context. Runs verification (lint, typecheck, test) and retries on failure.

Phase C
🔎

Review

3 specialist reviewers audit in parallel: correctness, security, and maintainability. All must approve before commit.

Loops until all tasks complete or budget is exhausted.

7 Perspectives, One Solution

Each persona brings a different priority. What they converge on is the best approach.

🛡

conservative

Stability, proven patterns, comprehensive error handling

➖

minimalist

Least code possible, eliminate unnecessary abstractions

🧪

tdd

Tests first, edge case coverage, test-driven design

⚡

performance

Algorithm efficiency, bundle size, rendering speed

👤

ux

Loading states, error messages, accessibility

🔒

security

Input validation, auth boundaries, XSS/CSRF prevention

🔧

pragmatist

Ship working software, practical trade-offs

Trivial tasks skip personas entirely. Standard tasks use 3. Complex tasks use all 7.

Key Features

📚

Cross-Task Learning

Review feedback and failure patterns propagate to subsequent tasks, preventing repeated mistakes.

🛡

Production-Ready

Auto-retry with fallback plans, smart circuit breaker, exponential backoff for transient errors.

💰

Budget Control

Per-persona, per-task, and total budget caps in USD. Automatic shutdown on exhaustion.

▶

Interrupt & Resume

Ctrl+C anytime. Resume with --resume. State is always persisted.

📦

Zero Setup

No npm install needed. Just Bun runtime + Claude Code CLI. Nothing else.

👁

Fine-Grained Control

--review to inspect task queue, --dry-run for plans only, --refine to adjust.

Quick Start

bash 
# Install
curl -fsSL https://raw.githubusercontent.com/kenpal-inc/convergent/main/install.sh | bash
bash 
# Run on your project
convergent \
  --context "src/,docs/" \
  --goal "Implement user authentication with JWT tokens"
bash 
# Review → Refine → Execute workflow
convergent --context "src/" --goal "Fix type errors" --review
convergent --refine "Remove task-001, add E2E tests to task-003"
convergent --resume
