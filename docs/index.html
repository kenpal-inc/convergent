Why "Convergent"?

In nature, when unrelated species independently arrive at the same solution, that solution is likely optimal. We built a tool around that principle.

Six unrelated plant species independently evolved the same molecule. When different paths converge on the same answer, that answer is likely optimal.

convergent applies this directly: strategies with different philosophies implement the same task in parallel git worktrees, each verified objectively (typecheck, lint, tests). An AI judge then analyzes all implementations — not to pick a winner, but to find where they converge. The shared design decisions become the solution.

How It Works

Give it a codebase and a goal. It handles the rest.

Phase 0
🗒

Task Generation

Analyzes your codebase and goal, decomposes into a structured task queue with dependency graph. Assigns types: code, explore, or command.

Phase T
⚔

Convergent Synthesis

N strategies implement independently in parallel git worktrees. Each is verified objectively (typecheck, lint, tests). An AI judge identifies convergent patterns and synthesizes the optimal solution.

Phase C
🔎

Review

3 specialist reviewers audit in parallel: correctness, security, and maintainability. All must approve before commit.

Loops until all tasks complete or budget is exhausted.

Diverse Strategies

Each competitor implements the same task with a different philosophy. Where they converge, that's the answer.

🔧

pragmatist

Follow the plan precisely. No deviations, no extras. Ship exactly what was specified.

🛡

thorough

Comprehensive implementation with defensive validation, edge case handling, and robust error recovery.

🔍

deconstructor

Challenge assumptions. Question the plan, find better approaches, refactor when warranted.

Trivial tasks use 1 strategy. Standard and complex tasks run a full convergence round.

Key Features

📚

Cross-Task Learning

Review feedback and failure patterns propagate to subsequent tasks, preventing repeated mistakes.

🛡

Production-Ready

Auto-retry with fallback plans, smart circuit breaker, exponential backoff for transient errors.

💰

Budget Control

Per-task and total budget caps in USD. Automatic shutdown on exhaustion.

▶

Interrupt & Resume

Ctrl+C anytime. Resume with --resume. State is always persisted.

📦

Zero Setup

No npm install needed. Just Bun runtime + Claude Code CLI. Nothing else.

👁

Fine-Grained Control

--review to inspect task queue, --dry-run for plans only, --refine to adjust.

When to Use

Convergence-based development shines in specific scenarios.

🎯

Multiple Valid Approaches

When requirements are ambiguous and design decisions matter. "Add real-time notifications" — WebSocket vs SSE vs polling? Let competitors explore different paths.

🔄

Large Refactoring

Breaking changes with multiple migration strategies. Parallel worktrees let each approach run safely. The one that passes verification wins.

✅

Well-Tested Codebases

Existing test suites create strong selection pressure. Bad implementations are eliminated automatically by objective verification.

🌱

Greenfield Features

New features in existing projects where competitors independently make architectural decisions. Convergence signals which choices are natural.

Cost Profile (real-world benchmark)

Task: CSRF implementation on a Hono + React production codebase

Tasks generated:    5-6
Convergence rounds: 4
Total cost:         ~$25-35
Wall clock time:    ~60 min
Strategies/round:   1 (trivial) / 2 (standard) / 3 (complex)

Quick Start

bash 
# Install
curl -fsSL https://raw.githubusercontent.com/kenpal-inc/convergent/main/install.sh | bash
bash 
# Run on your project
convergent \
  --context "src/,docs/" \
  --goal "Implement user authentication with JWT tokens"
bash 
# Review → Refine → Execute workflow
convergent --context "src/" --goal "Fix type errors" --review
convergent --refine "Remove task-001, add E2E tests to task-003"
convergent --resume
