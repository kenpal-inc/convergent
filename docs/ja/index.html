なぜ "convergent" か

自然界では、無関係な種がそれぞれ独立して同じ解にたどり着くことがあります。この原理をそのままツールにしました。

6つの無関係な植物種が、独立して同じ分子を進化させました。異なる道筋が同じ答えに収斂する。それは、おそらく最適解であることを示しています。

convergent はこの原理をそのまま実践します。異なる哲学を持つ複数の戦略が並列のgit worktreeで同じタスクに挑み、それぞれが客観的に検証（typecheck, lint, tests）されます。AI審判は勝者を選ぶのではなく、全実装に共通する設計判断を特定し、そこから最適な解を合成します。

How it works

コードベースとゴールを渡すだけで構いません。あとは自動で進行します。

Phase 0
🗒

Task Generation

コードベースとゴールを解析し、依存関係グラフ付きのタスクキューを生成します。タスク種別: code, explore, command

Phase T
⚔

Convergent Synthesis

N個の戦略が並列のgit worktreeで独立して実装します。それぞれが客観的に検証（typecheck, lint, tests）された後、AI審判が全実装の収斂パターンを分析し、最適な解を合成します。

Phase C
🔎

Review

3人の専門レビュアーが並列に監査します。正確性、セキュリティ、保守性の観点から評価します。全員が承認するまでコミットされません。

Phase F
🔨

Integration Check

全タスク完了後、AIがタスク横断の整合性を検証します。未実装のAPIルート、壊れたimport、スキーマの不整合を検出し、重大な問題は自動修正します。

全タスク完了または予算消化まで繰り返します。Phase F は最後に1回実行されます。

Diverse Strategies

各コンペティターは異なる哲学で同じタスクを実装します。収斂する設計判断が、答えになります。

🔧

pragmatist

プランに忠実に実装します。逸脱なし、余計なものなし。指定された通りに出荷します。

🛡

thorough

防御的バリデーション、エッジケース対応、堅牢なエラー回復を含む包括的な実装。

🔍

deconstructor

前提を疑い、より良いアプローチを見つけ、必要に応じてリファクタリングします。

簡単なタスクは1戦略。標準・複雑なタスクはフル収斂ラウンドを実行します。

Key Features

📚

Cross-Task Learning

レビューのフィードバックと失敗パターンが後続タスクに伝播します。

🛡

Production-Ready

フォールバック付き自動リトライ、スマートサーキットブレーカー、指数バックオフに対応しています。

💰

Budget Control

タスク単位・全体の予算上限（USD）を設定できます。消化時に自動停止します。

▶

Interrupt & Resume

いつでも Ctrl+C で停止できます。--resume で再開できます。状態は常に保存されます。

📦

Zero Setup

npm install は不要です。Bun ランタイム + Claude Code CLI のみで動作します。

👁

Fine-Grained Control

--review でタスクキューを確認できます。--dry-run でプランのみを実行できます。--refine で調整できます。

いつ使うべきか

収斂方式の開発が活きるシナリオがあります。

🎯

正解が複数あるとき

要件が曖昧で設計判断が重要な場面。「リアルタイム通知を追加」— WebSocket vs SSE vs ポーリング？コンペティターに異なるアプローチを探索させましょう。

🔄

大規模リファクタリング

破壊的変更を含むマイグレーション。並列のworktreeで各アプローチを安全に試行できます。検証を通過したものが勝利します。

✅

テストが充実したコードベース

既存のテストスイートが強い選択圧として機能します。不適切な実装は客観的な検証で自動的に排除されます。

🌱

既存プロジェクトへの新機能追加

コンペティターが独立してアーキテクチャを決定します。収斂シグナルがどの選択が自然かを示します。

コストプロファイル（実測値）

課題: Hono + React 本番コードベースへのCSRF実装

生成タスク数:         5-6
収斂ラウンド実行回数:  4
合計コスト:           ~$25-35
実行時間:             ~60分
戦略数/ラウンド:      1 (trivial) / 2 (standard) / 3 (complex)

クイックスタート

bash 
# インストール
curl -fsSL https://raw.githubusercontent.com/kenpal-inc/convergent/main/install.sh | bash
bash 
# プロジェクトで実行
convergent \
  --context "src/,docs/" \
  --goal "Implement user authentication with JWT tokens"
bash 
# レビュー → 修正 → 実行 ワークフロー
convergent --context "src/" --goal "Fix type errors" --review
convergent --refine "Remove task-001, add E2E tests to task-003"
convergent --resume

Full Transparency

すべての判断がログに残ります。すべてのコンペティター出力を検査できます。

├── latest → runs/2026-02-12T20-30-00
└── runs/
    └── 2026-02-12T20-30-00/
        ├── tasks.json
        ├── state.json
        ├── budget.json
        ├── learnings.json
        ├── reports/
        │   ├── summary.md
        │   └── task-001.md
        └── logs/
            ├── orchestrator.log
            └── task-task-001/
                ├── competitor-0.log
                ├── competitor-1.log
                ├── judge.json
                ├── verify.log
                └── review.json

--resume で再開できます。失敗タスクは自動ロールバックされます。何も失われません。
