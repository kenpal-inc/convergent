convergent

Claude Code 向け自律開発オーケストレーター

複数のAIペルソナが独立して解決策を設計し、最適な実装へと収斂させます。それは、カフェインが6つの無関係な植物種で独立に進化したのと同じ現象です。

無関係な種が、それぞれ独立して同じ解にたどり着くことがあります。

6つの無関係な植物種が、独立して同じ分子を進化させました。異なる道筋が同じ答えに収斂する。それは、おそらく最適解であることを示しています。

convergent は、この原理をコードに応用します。異なる優先度を持つ複数のAIペルソナが同じ問題を分析します。合意した要素のみを抽出し、シンセサイザーが統合します。

How it works

コードベースとゴールを渡すだけで構いません。あとは自動で進行します。

Phase 0
🗒

Task Generation

コードベースとゴールを解析し、依存関係グラフ付きのタスクキューを生成します。タスク種別: code, explore, command

Phase A
🧸

Convergence

3〜7のペルソナが、異なる優先度で並列に設計します。合意度が十分に高ければ早期に終了します。シンセサイザーが最適なプランへ統合します。

Phase B
⌨

Execution

収斂したプランを実行します。タスク間の学習をコンテキストとして活用します。検証（lint, typecheck, test）を実行し、失敗した場合はリトライします。

Phase C
🔎

Review

3人の専門レビュアーが並列に監査します。正確性、セキュリティ、保守性の観点から評価します。全員が承認するまでコミットされません。

全タスク完了または予算消化まで繰り返します。

7 Perspectives, One Solution

各ペルソナは異なる優先度を持っています。収斂した結論が最良のアプローチです。

🛡

conservative

安定性、実績あるパターン、包括的なエラーハンドリング

➖

minimalist

最小限のコード、不要な抽象化の排除

🧪

tdd

テストファースト、エッジケースの網羅

⚡

performance

アルゴリズム効率、バンドルサイズ、描画速度

👤

ux

ローディング状態、エラーメッセージ、アクセシビリティ

🔒

security

入力バリデーション、認証境界、XSS/CSRF防止

🔧

pragmatist

動くソフトウェアを出荷する、実用的なトレードオフ

簡単なタスクはペルソナなし。標準は3つ。複雑なタスクは全7つを使用します。

Key Features

📚

Cross-Task Learning

レビューのフィードバックと失敗パターンが後続タスクに伝播します。

🛡

Production-Ready

フォールバック付き自動リトライ、スマートサーキットブレーカー、指数バックオフに対応しています。

💰

Budget Control

ペルソナ単位・タスク単位・全体の予算上限（USD）を設定できます。消化時に自動停止します。

▶

Interrupt & Resume

いつでも Ctrl+C で停止できます。--resume で再開できます。状態は常に保存されます。

📦

Zero Setup

npm install は不要です。Bun ランタイム + Claude Code CLI のみで動作します。

👁

Fine-Grained Control

--review でタスクキューを確認できます。--dry-run でプランのみを実行できます。--refine で調整できます。

クイックスタート

bash 
# インストール
curl -fsSL https://raw.githubusercontent.com/kenpal-inc/convergent/main/install.sh | bash
bash 
# プロジェクトで実行
convergent \
  --context "src/,docs/" \
  --goal "Implement user authentication with JWT tokens"
bash 
# レビュー → 修正 → 実行 ワークフロー
convergent --context "src/" --goal "Fix type errors" --review
convergent --refine "Remove task-001, add E2E tests to task-003"
convergent --resume

Full Transparency

すべての判断がログに残ります。すべてのペルソナ出力を検査できます。

.convergent/
├── latest → runs/2026-02-12T20-30-00
└── runs/
    └── 2026-02-12T20-30-00/
        ├── tasks.json
        ├── state.json
        ├── budget.json
        ├── learnings.json
        ├── reports/
        │   ├── summary.md
        │   └── task-001.md
        └── logs/
            ├── orchestrator.log
            └── task-001/
                ├── persona-conservative.json
                ├── persona-tdd.json
                ├── persona-security.json
                ├── synthesis.json
                ├── execution.log
                ├── verify.log
                └── review.json

--resume で再開できます。失敗タスクは自動ロールバックされます。何も失われません。
