convergent

Claude Code 向け自律開発オーケストレーター

複数のAIペルソナが独立して解決策を設計し、最適な実装に収斂する — カフェインが6つの無関係な植物種で独立に進化したように。

収斂進化とは？

無関係な種が独立して同じ解に辿り着くとき、その解はおそらく最適である。

6つの無関係な植物種が、独立して同じ分子を進化させた。異なる道が同じ答えに収斂するとき、その答えはおそらく最適解である。

convergent はこの原理をコードに応用する。異なる優先度を持つ複数のAIペルソナが同じ問題を分析し、合意した要素をシンセサイザーが統合する。

仕組み

コードベースとゴールを渡すだけ。あとは全自動。

Phase 0
🗒

タスク生成

コードベースとゴールを分析し、依存関係グラフ付きのタスクキューを生成。タスク種別: code, explore, command。

Phase A
🧸

収斂進化

3〜7のペルソナが異なる優先度で並列に設計。高い合意で早期終了。シンセサイザーが最適プランに統合。

Phase B
⌨

実装

収斂したプランを実行。タスク間の学習をコンテキストとして活用。検証（lint, 型チェック, テスト）を実行し、失敗時はリトライ。

Phase C
🔎

コードレビュー

3人の専門レビュアーが並列に監査: 正確性、セキュリティ、保守性。全員が承認するまでコミットされない。

全タスク完了または予算消化まで繰り返す。

7つの視点、1つの最適解

各ペルソナが異なる優先度を持つ。収斂した部分が最良のアプローチ。

🛡

conservative

安定性、実績あるパターン、包括的なエラーハンドリング

➖

minimalist

最小限のコード、不要な抽象化の排除

🧪

tdd

テストファースト、エッジケースの網羅

⚡

performance

アルゴリズム効率、バンドルサイズ、描画速度

👤

ux

ローディング状態、エラーメッセージ、アクセシビリティ

🔒

security

入力バリデーション、認証境界、XSS/CSRF防止

🔧

pragmatist

動くソフトウェアを出荷する、実用的なトレードオフ

簡単なタスクはペルソナなし。標準は3つ。複雑なタスクは全7つを使用。

主な機能

📚

タスク間学習

レビューのフィードバックと失敗パターンが後続タスクに伝播し、同じミスを防ぐ。

🛡

耐障害性

フォールバック付き自動リトライ、スマートサーキットブレーカー、指数バックオフ。

💰

予算管理

ペルソナ単位・タスク単位・全体の予算上限（USD）。消化時に自動停止。

▶

レジューム

いつでも Ctrl+C。--resume で再開。状態は常に保存される。

📦

ゼロ依存

npm install 不要。Bun ランタイム + Claude Code CLI だけ。

👁

実行前レビュー

--review でタスクキュー確認、--dry-run でプランのみ、--refine で調整。

クイックスタート

bash 
# インストール
curl -fsSL https://raw.githubusercontent.com/kenpal-inc/convergent/main/install.sh | bash
bash 
# プロジェクトで実行
convergent \
  --context "src/,docs/" \
  --goal "Implement user authentication with JWT tokens"
bash 
# レビュー → 修正 → 実行 ワークフロー
convergent --context "src/" --goal "Fix type errors" --review
convergent --refine "Remove task-001, add E2E tests to task-003"
convergent --resume

完全な透明性

すべての判断がログに残る。すべてのペルソナ出力が検査可能。

├── latest → runs/2026-02-12T20-30-00

--resume で再開。失敗タスクは自動ロールバック。何も失われない。
